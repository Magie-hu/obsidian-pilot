# Obsidian Pilot - Free vs Paid Feature Layers

## Product Positioning
Bidirectional bridge between LLM-Wiki and Obsidian
Target users:
1. Developers using Hermes/Codex for local AI knowledge bases
2. Obsidian double-link power users who want AI to auto-organize notes

## Free Open-Source Core (GitHub Public)

### Basic Features
- Auto-initialize LLM-Wiki standard directory structure (`raw/`, `wiki/`)
- One-click vault initialization (folders + templates + index)
- Note import and auto-classification (keyword matching)
- Link automation (detect missing links, update index)
- Basic sync: bidirectional local sync, no duplicates, deduplication
- Auto-generate `[[double-links]]`

### Free Tier Limitations
- Local only, no cloud dependency
- Basic classification algorithm (keyword matching)
- No team collaboration
- No advanced visualization

## Paid Pro Features (Closed Source)

### Pro Features
- LLM-powered classification (integrates with large language models)
- Knowledge graph visualization (SVG/PNG export)
- Team collaboration (multi-user, permission management)
- Cloud sync
- Custom template engine
- Advanced analytics (stats, activity analysis, trend prediction)
- Plugin marketplace

### Enterprise Features
- SSO (LDAP/AD integration)
- Audit logs
- API access
- Custom branding

## Key Pitfalls to Avoid
1. **Code isolation:** Separate open-source core and paid modules into different repos/folders to avoid MIT license disputes
2. **No forced payment:** Basic sync, directory generation, double-link basics remain free forever
3. **Clear positioning:** README must clarify this is a **bridge tool**, not LLM-Wiki itself — no copyright conflict
4. **No exaggeration:** No binding to any closed-source LLM, all local file operations, privacy-first selling point

## Unique Competitive Advantage

Market has two types of tools:
1. **Obsidian plugins:** Note management only, no Hermes/Codex local AI integration
2. **LLM Agent scripts:** File read/write only, no Obsidian graph or Templater depth

We are the **only vertical tool bridging both sides** — precise user base, minimal competition, high willingness to pay for time-saving automation.
