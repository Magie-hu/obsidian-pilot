# Obsidian 第一助手 — 项目规划书

> 版本：v1.0 | 日期：2026-07-09 | 作者：宁小瓣

---

## 一、产品定位

**Obsidian 第一助手** — 面向 Obsidian 初学者的 AI 管理助手，帮助用户从零基础到熟练使用，实现笔记自动化整理、知识图谱构建、AI 助手智能路由。

### 核心价值

1. **降低门槛** — 新手无需理解 Obsidian 的复杂概念，一键完成初始化
2. **节省成本** — 智能路由减少用户与 Hermes/Codex/OpenClaw 等 AI 助手的 token 消耗
3. **知识沉淀** — 自动构建知识图谱，让笔记从"散乱的 markdown"变成"可检索的知识库"

### 目标用户

| 用户群 | 痛点 | 解决方式 |
|--------|------|----------|
| 程序员/开发者 | 技术笔记散乱、找不到 | 自动分类 + 关键词链接 |
| 内容创作者 | 素材收集无序、过期内容堆积 | 每日自动整理 + 过期提醒 |
| 知识工作者 | 笔记量大、关联弱 | 知识图谱 + 孤立笔记检测 |

## 二、核心功能

### 2.1 新手引导（第一阶段）

- **一键初始化知识库**
  - 预设 5 套文件夹模板（技术笔记、项目管理、读书笔记、生活记录、综合型）
  - 自动创建 Templater 模板（frontmatter、标题格式、标签规范）
  - 生成 00-总索引.md，包含所有分类链接
- **插件推荐引擎**
  - 根据用户初始选择推荐插件（如：写技术笔记→推荐 Dataview）
  - 每个推荐带一句话说明："这个插件能帮你 XXX"
  - 引导安装并配置

### 2.2 笔记导入（第二阶段）

- **自动分类**
  - 扫描所有 .md 文件，根据内容关键词分配到对应文件夹
  - 示例：检测到"Kubernetes"→`服务器运维/`
- **自动加标签**
  - 基于内容语义分配标签
  - 示例：检测到"AI Agent"→`AI智能体`
- **自动补 frontmatter**
  - 提取日期、来源、相关笔记
  - 统一格式：tags、date、sources、related

### 2.3 链接自动化（第三阶段）

- **关键词自动链接**
  - 笔记中提到"Claude Code"，自动链接到已有的 `[[Claude Code vs Codex CLI]]`
  - 基于 TF-IDF 或 Word2Vec 计算内容相似度
- **孤立笔记检测**
  - 扫描所有笔记，找出没有任何双向链接的"孤岛"笔记
  - 提醒用户补充关联
- **索引页自动维护**
  - 新增笔记后自动更新 00-总索引.md
  - 支持手动确认机制（输出变更建议，不直接改文件）

### 2.4 AI 助手智能路由（第四阶段）

- **本地知识库优先**
  - 用户提问时，先检查 Obsidian 知识库有没有相关内容
  - 如果有，直接从知识库提取答案，不需要调用 AI 模型
  - 示例：用户问"帮我写一个 Flask 路由"→助手先搜知识库，找到 Flask 笔记，直接给出答案
- **上下文压缩**
  - 把知识库里的相关内容提取出来，压缩成精简的上下文
  - 再发给 AI 助手，减少 token 用量
  - 示例：用户问"帮我优化服务器配置"→助手从知识库提取 Nginx 优化笔记，压缩成 500 token 发给 AI
- **跨助手协调**
  - 统一管理 Hermes、Codex、OpenClaw 的配置和配额
  - 推荐最省 token 的方案

### 2.5 日常维护（第五阶段）

- **每日自动整理**
  - 更新索引页
  - 检测孤立笔记
  - 标记过期内容（超过 30 天的热点→建议归档）
  - 发现重复笔记→提醒合并
- **RSS 集成**
  - 与 RSS 搜集脚本联动
  - 自动写入时加标签、分类、建索引

## 三、技术架构

### 3.1 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 语言 | Python 3.11+ | 本地脚本，不需要 Docker |
| 知识库读取 | pathlib + re | 读取 .md 文件，正则提取 frontmatter |
| 内容相似度 | scikit-learn (TF-IDF) | 关键词匹配做分类和链接 |
| 知识图谱 | NetworkX | 生成笔记关系图 |
| 定时任务 | cronjob (Hermes Agent) | 每日自动运行 |
| AI 集成 | OpenAI API / 火山引擎 API | 智能路由和上下文压缩 |

### 3.2 目录结构

```
obsidian-assistant/
├── src/
│   ├── init.py          # 新手引导 + 一键初始化
│   ├── import.py        # 笔记导入 + 自动分类
│   ├── link.py          # 链接自动化
│   ├── route.py         # AI 助手智能路由
│   ├── maintain.py      # 日常维护
│   └── utils.py         # 工具函数
├── templates/
│   ├── frontmatter.md   # frontmatter 模板
│   ├── index.md         # 索引页模板
│   └── hot_news.md      # 热点新闻模板
├── config.yaml          # 配置文件
└── README.md            # 使用说明
```

### 3.3 工作流程

```
用户提问
  ↓
检查知识库是否有相关内容
  ↓
有 → 直接从知识库提取答案
  ↓
没有 → 压缩知识库内容 + 用户问题 → 发给 AI 助手
  ↓
AI 返回答案 → 存入知识库 → 下次可直接检索
```

## 六、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 笔记格式不统一 | 分类准确率下降 | 提供格式转换工具 |
| AI API 调用延迟 | 智能路由响应慢 | 本地缓存 + 超时降级 |
| 用户数据丢失 | 信任危机 | 变更前备份 + 手动确认机制 |
| 功能过于复杂 | 用户流失 | 分阶段发布，先 MVP 后迭代 |
